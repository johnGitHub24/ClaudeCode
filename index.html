Step 1

安裝 Claude Code

支援 Windows、macOS、Linux，需要 Node.js 18+ 環境。

🪟 Windows

# 方法一：npm 安裝（推薦）
$ npm install -g @anthropic-ai/claude-code

# 方法二：WSL 環境
$ wsl --install
$ npm install -g @anthropic-ai/claude-code
⚠️ Windows 建議使用 WSL2 以獲得最佳體驗

🍎 macOS

# 先確認 Node.js 版本
$ node -v # 需要 v18+

# 安裝 Claude Code
$ npm install -g @anthropic-ai/claude-code

# 驗證安裝
$ claude --version

🐧 Linux

# Ubuntu/Debian
$ curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -
$ sudo apt-get install -y nodejs
$ npm install -g @anthropic-ai/claude-code

🔑 設定 API Key

# 首次執行會引導登入
$ claude

# 或手動設定環境變數

進階技巧

10 個提升效率的技巧

這些技巧能讓你的 Claude Code 使用體驗提升到另一個層次。

1

使用 CLAUDE.md 記錄規範

在專案根目錄建立 CLAUDE.md，記錄編碼規範、禁止行為、常用指令，每次對話都會自動載入。

2

管道串接提升自動化

用 cat error.log | claude -p "分析錯誤原因" 將其他工具輸出直接傳給 Claude 分析。

3

善用 /compact 節省 token

長對話後執行 /compact，Claude 會自動摘要對話並保留重要上下文，延長可用 token。

4

先規劃後執行

複雜任務先問「你打算怎麼做？」，確認計畫無誤再說「好，請執行」，避免走錯方向。

5

使用 ! 執行 shell 指令

在 REPL 中輸入 ! git log --oneline -5 可直接執行 shell 指令，結果會顯示在對話中供 Claude 參考。

6

設定 .claudeignore

排除 node_modules、大型資料檔、敏感設定，讓 Claude 聚焦在真正重要的程式碼上。

7

Hooks 自動執行 Lint

設定 PostToolUse hook，每次 Claude 編輯檔案後自動執行 ESLint 或 Prettier，維持程式碼品質。

8

多目錄開發

使用 /add-dir ../backend 加入相關目錄，讓 Claude 在 monorepo 中跨目錄操作。

9

Opus 模型處理複雜推理

架構設計、複雜 bug 用 /model claude-opus-4-7；簡單任務用預設 Sonnet 節省費用。

10

CI/CD 非互動模式

在 CI 管道中使用 claude -p --allowedTools "Read,Grep" "審查PR" 限制可用工具確保安全。

自動化

Hooks 設定範例

Hooks 讓你在 Claude 執行工具前後自動觸發自訂腳本。

PostToolUse — 自動執行 Lint

每次 Claude 編輯檔案後自動執行 ESLint 檢查。

// .claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{
          "type": "command",
          "command": "npx eslint {{file}}"
        }]
      }
    ]
  }
}

PreToolUse — 禁止危險操作

在 Claude 執行 Bash 指令前攔截並記錄危險操作。

// .claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{
          "type": "command",
          "command": "./scripts/audit-cmd.sh"
        }]
      }
    ]
  }
}

Stop Hook — 完成通知

Claude 完成任務時發送系統通知，適合長時間任務。

// .claude/settings.json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [{
          "type": "command",
          "command": "notify-send 'Claude 任務完成'"
        }]
      }
    ]
  }
}

CLAUDE.md — 讓 Claude 記住你的專案規則

CLAUDE.md 是 Claude 每次啟動都會自動讀取的記憶檔，相當於給 AI 的「員工手冊」。

完整 CLAUDE.md 範本

# CLAUDE.md — 專案 AI 助理規範

## 專案概述
電商平台後端 API，Node.js + TypeScript + PostgreSQL。
目前 Sprint：結帳流程重構（截止 2026-05-15）。

## 技術棧
- Runtime：Node.js 20 + TypeScript 5.x
- 框架：Fastify（不是 Express）
- ORM：Prisma（不要用 raw SQL）
- 測試：Vitest + Supertest
- CI：GitHub Actions

## 程式碼規範
- 所有函式必須有 JSDoc 型別標注
- 錯誤處理用 Result<T, E> 模式，不用 throw
- 禁止 any，使用 unknown 再 narrow
- 命名：camelCase 變數，PascalCase 型別

## 禁止行為
- 不可直接修改 package.json
- 不可 git push 到 main 分支
- 不可刪除 migrations/ 下的任何檔案
- 不可使用 console.log（用 logger.*）

## 常用指令
- 啟動開發：`npm run dev`
- 執行測試：`npm test`
- 生成 migration：`npx prisma migrate dev`
- 型別檢查：`npx tsc --noEmit`

多層 CLAUDE.md 架構

可以在子目錄放置 CLAUDE.md，覆寫或補充根目錄的規則。

project-root/
├── CLAUDE.md ← 全專案規範
├── src/
│ ├── api/
│ │ └── CLAUDE.md ← API 模組特定規則
│ └── db/
│ └── CLAUDE.md ← 資料庫操作規則
└── tests/
    └── CLAUDE.md ← 測試撰寫規範

  """單一 Subagent：執行指定任務"""

  while True:

      model="claude-sonnet-4-6", max_tokens=4096,
      tools=tools, messages=messages

      return resp.content[0].text
    # ... 處理 tool_use

async def orchestrator(project_path: str):
  """並行派發子任務"""
  tasks = [
    subagent(f"分析 {project_path}/src 的安全漏洞", security_tools),
    subagent(f"分析 {project_path}/src 的效能問題", perf_tools),

  ]

  return summarize(results)

Hook stdin 輸入格式

Hook 腳本透過 stdin 接收工具呼叫的 JSON 資訊。

#!/bin/bash # pre-bash.sh
INPUT=$(cat) # 讀取 stdin JSON

# JSON 格式：
# { "tool": "Bash", "input": { "command": "rm -rf /" } }

COMMAND=$(echo "$INPUT" | python3 -c "import sys,json; print(json.load(sys.stdin)['input']['command'])")

# 禁止危險指令
if echo "$COMMAND" | grep -qE "rm -rf|sudo rm|drop table"; then
  echo "BLOCKED: 危險指令被攔截" >&2
  exit 2 # exit 2 = 阻止工具執行
fi

Hook 退出碼說明

退出碼 行為
0 成功，繼續執行工具
1 警告，顯示 stderr 給 Claude，仍繼續
2 阻止，取消工具執行並回報錯誤
ℹ️ PostToolUse Hook 的退出碼 2 會讓 Claude 知道後處理失敗，可觸發重試。

完整實戰腳本：自動格式化 + Lint

每次 Claude 寫入檔案後自動執行格式化與 Lint，保持程式碼整潔。

#!/bin/bash # scripts/post-write.sh
INPUT=$(cat)
FILE=$(echo "$INPUT" | python3 -c
  "import sys,json; d=json.load(sys.stdin); print(d.get('input',{}).get('file_path',''))")

if [ -z "$FILE" ]; then exit 0; fi

  *.ts|*.tsx|*.js|*.jsx)
    npx prettier --write "$FILE" 2>/dev/null
    npx eslint --fix "$FILE" 2>/dev/null
    ;;
  *.py)
    python3 -m black "$FILE" 2>/dev/null
    python3 -m ruff check --fix "$FILE" 2>/dev/null
    ;;
esac
exit 0

自訂 Subagent — 打造可重用 AI 角色

將程式碼審查員、安全稽核師、文件撰寫員等角色封裝成 Subagent，讓 Claude 在適當時機自動委派任務，保護主對話的 context 不被浪費。

Subagent 是什麼？

擁有獨立系統提示、工具集、權限設定的特化 AI 助手。主對話把任務委派給 subagent，subagent 在自己的 context 視窗內工作，只回傳結果摘要——不污染主對話的 token。

何時使用？

某個側任務會把大量搜尋結果、Log 或檔案內容倒入主對話，但這些內容之後不會再被引用 → 交給 subagent 在獨立視窗處理，保留主對話 context。

內建 Subagents

Explore（Haiku，唯讀）— 搜尋程式碼庫
Plan（繼承，唯讀）— Plan Mode 研究
general-purpose（繼承，全工具）— 複雜多步任務
Claude 會自動在合適時機使用這些 agents。

Claude Code 15 大專業技巧

Claude Code 創作者 Boris Cherny 親自揭露的 15 個高頻實用技巧，快速突破使用瓶頸。

1

手機直接寫程式（Mobile Programming）

在 iOS / Android 版 Claude App 的左側欄切換到 Code 分頁，可直接在手機上撰寫並執行程式碼，隨時隨地不間斷開發。

2

跨裝置接力（Cross-Device Handoff）

用 claude --teleport 或 REPL 中輸入 /teleport 將目前工作階段傳送到另一台裝置；輸入 /remote-control 可開啟遠端控制模式。

3

排程執行（/loop 與 /schedule）

用 /loop 5m 任務描述 讓 Claude 每 5 分鐘自動執行一次；用 /schedule 安排最多一週後才執行的延遲任務，打造自動化工作流。

4

Hooks 掌控 Agent 生命週期

在 SessionStart、PreToolUse、PermissionRequest、Stop 四個關鍵時機插入自訂邏輯，確定性地管控 Agent 行為，而非依賴 prompt。

5

Cowork 遠端桌面派發

透過 Claude 的安全介面從任何裝置遠端操控桌面電腦，管理 MCP 伺服器、瀏覽器與本機應用程式，實現跨裝置任務派發。

6

Chrome 擴充功能驗證前端

安裝 Claude Code Chrome Extension，讓 Claude 直接看到瀏覽器渲染結果，無需手動截圖即可迭代 UI 修改，大幅加速前端開發回饋循環。

7

自動啟動並測試 Web Server

Claude Desktop 內建瀏覽器，會自動偵測並啟動你的 Web Server，直接在內建瀏覽器中測試，無需手動操作，開發效率大增。

8

工作階段分支（Session Forking）

用 /branch 或 claude --resume <id> --fork-session 從同一個任務分叉出不同方向，同時探索多種解法，找到最佳方案後再合併。

9

中途插問（/btw）

在 Agent 執行長任務時，輸入 /btw 你的問題 可插入快問快答，不打斷 Agent 的主線工作。例如：/btw 這邊用 async/await 還是 Promise 比較好？

10

Git Worktrees 平行實例

用 git worktree add ../feature-branch 建立獨立工作目錄，同時跑數十個 Claude 實例，每個都在獨立目錄作業，互不干擾，極大化平行開發效率。

11

大規模平行遷移（/batch）

用 /batch 將大型程式碼庫遷移任務分散到數百個 worktree Agent 同時執行，大規模並行修改讓原本需要幾天的重構在數小時內完成。

12

--bare 快速啟動（跳過自動載入）

加上 --bare 旗標跳過 CLAUDE.md 與上下文自動載入，啟動速度可提升最多 10 倍，非常適合非互動式自動化腳本與 CI/CD 環境。

13

跨 Repo 存取（--add-dir）

啟動時加上 claude --add-dir ../other-repo 即可在單一工作階段內存取並修改多個 Git 倉庫，完美支援 monorepo 或前後端分離專案。

14

自訂 Agent 角色（--agent）

在 .claude/agents/ 目錄放置 Markdown 檔案定義可重用角色（程式碼審查員、資安稽核師）：設定專屬系統提示與工具組，用 claude --agent reviewer 一鍵切換。

15

語音輸入寫程式（/voice）

在 REPL 輸入 /voice 或在 Desktop / iOS 設定中開啟語音模式，用說話方式下指令，解放雙手、提升輸入速度，讓思路更流暢地轉化為程式碼。
